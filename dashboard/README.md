# Pursuits dashboard

Visual view of the pursuits node tree. **The wiki is the source of truth** — the dashboard is generated by walking `pursuits/`. Tree ↔ wiki are the same structure by construction (no drift).

## Two ways to open

- **App mode (full features — add notes, open/edit docs):** run the local server, then visit the URL.
  ```bash
  node dashboard/serve.js        # http://localhost:8777  (rebuilds data on start + after each add)
  ```
- **Static mode (read-only view):** `open dashboard/index.html`. Tree + navigation work; the **＋ Add note** button and in-place doc opening are disabled (the doc icon falls back to opening the raw `.md` in the browser).

## Using it

- **Doc icon** (on every node) → opens an **inline editor** (right drawer): edit the doc and **Save** (or ⌘/Ctrl-S) writes it straight back to the file; **Open in app** opens it externally. (Static mode has no editor — the doc icon just opens the raw `.md` read-only.)
- **＋ on hover** over a node → add a child **note**; **＋ Add note** in the header → add a new top-level pursuit. A note creates a real wiki doc (`status: note`) and wires it in as a child; the view refreshes immediately.
- **Gear icon** (above the doc icon) → size dialog with ▲/▼ to grow/shrink that node **and its whole subtree**. Sizes persist per browser via `localStorage` (view-only — not written to the wiki).
- **Drag a node** → repositions that node **and its children** (offset cascades to the subtree); dragging **empty space** still pans the canvas. Positions persist per browser via `localStorage` (view-only). A small click still expands/collapses or opens the doc.

Size and position are saved immediately and keyed by a **stable per-node `nid`** (in each node's frontmatter), so they survive renames and leaf→branch conversions — adding a child to a sized note no longer resets it.
- Click a **branch** to expand/collapse · scroll = zoom · drag = pan · `#` badge = hidden children · Buttons: Expand all / Collapse / Fit.

## Source of truth → how it maps

Each tree node is a wiki page under [`pursuits/`](../pursuits):

- **Forest root:** `pursuits/index.md` — `children:` lists the pursuit roots (order = display order).
- **Branch node:** `<slug>/index.md` with a `children: [slug, …]` list.
- **Leaf node:** `<slug>.md` (no children).
- Page **title** (frontmatter `title:`, else first `# H1`) = the node label. Folder path = `url` the node opens.

## Change the tree

- **Edit a node:** edit its page in `pursuits/…`.
- **Add a node:** create the page (leaf `<slug>.md`, or branch `<slug>/index.md`) and add its `slug` to the parent's `children:`.
- **Reorder / remove:** edit the parent's `children:` list.
- **Regenerate the view:**
  ```bash
  node dashboard/build.js   # walks pursuits/ -> dashboard/data.js  (warns on missing/broken children)
  ```

## Files

`index.html` + `app.js` (D3 viewer + add-note UI) · `data.js` (generated) · `build.js` (wiki→data) · `vendor/d3.min.js` (vendored, offline) · `serve.js` (local **app** server: static + `/api/ping`, `/api/open`, `/api/add`).

## API (app mode)

- `GET /api/ping` → `{ok}` (used to detect app vs static mode)
- `GET /api/open?path=pursuits/…md` → opens the doc in your default app (macOS `open`); path restricted to `pursuits/`
- `GET /api/doc?path=pursuits/…md` → `{content, title}` for the inline editor
- `POST /api/save {path, content}` → writes the doc back, rebuilds `data.js`, returns `{title, trees}` (tree relabels if the `title:` changed)
- `POST /api/add {parent, title, note}` → creates the node doc, wires it into the parent's `children:`, rebuilds `data.js`, returns the fresh forest. A leaf parent is auto-promoted to a branch (`<slug>/index.md`).
